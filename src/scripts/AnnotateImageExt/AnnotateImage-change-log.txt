2026/03/03 - v2.3.0

- Ported to the new V8 JavaScript runtime on PixInsight 1.9.4 Lockhart.

- Complete revision and reimplementation of critical user interface features:

  * Improved annotation preview interface with significantly faster image
    renditions, arbitrary zoom factors in the [-100,+100] range (as in core
    image views), reduced memory requirements, and a new 'Zoom to Fit' feature.

  * Improved layer management functions in the script's main dialog. The Layers
    TreeBox supports multiple selections for moving and removing layers and
    many small bugs have been fixed in these operations.

  * Improved main dialog layout. The Layers list no longer changes its
    dimensions when different layers are selected.

  * The label placement optimization routine shows a message box to give the
    user a chance to abort time-consuming optimization tasks.

- Released under PCL License 2.0.

- PixInsight core version >= 1.9.4 required.

2024/02/28 - v2.2.11

- Improved robustness of test routines for object inclusion verification. On
  rare occasions, high-order surface splines can lead to false inclusions in
  coordinate transformations extrapolated far away from image boundaries. This
  can happen when checking visibility of arbitrarily selected objects such as
  planets, asteroids, and local catalogs. We perform an additional inverse
  transformation to validate object inclusion in these cases.

- New DE440 Kuiper Belt Objects (KBOs) annotation layer (30 objects).

- Magnitude limit controls are now available for the Asteroids layer.

2023/06/16 - v2.2.8

- Object annotations can now be defined and represented as rotated ellipses.

- Improved PGC2003, NGC/IC and Messier catalog support with galaxy aspect
  ratios and position angles. Galaxies are now represented as rotated ellipses
  when the necessary data are available.

- Local file catalogs (Named Stars, NGC/IC and Messier) have been improved with
  more accurate coordinates and magnitudes acquired from online database
  services. Local catalogs are now serialized in standard CSV format.

- Improved Lyra and Cygnus constellation shapes.

- New 'small size threshold' parameter. Representations of small objects can
  now be enlarged artificially to make them more visible in the annotation.
  This parameter is the radius in pixels of the largest elliptical annotations
  to which these artificial enlargement factors will be applied. Enlargement
  functions are logarithmic to achieve a smooth zooming effect without visible
  discontinuities. Set this parameter to zero to disable this feature and
  represent all objects with their (known) actual angular dimensions.

- Gaia XPSD catalog layers have a new parameter to select a coordinate
  precision in the 0-3 range (from arcseconds to milliarcseconds). This
  precision is the number of decimal digits shown for arcseconds in declination
  coordinate labels (an additional decimal digit is always shown for right
  ascension).

- Updates the new jsScriptInformation global variable.

- PixInsight core version >= 1.8.9-2 required.

2021/05/20 - v2.2.2

- Bugfix: Solved a rare numerical degeneracy problem in the construction of
  BRQuadTree structures used by the label placement optimization algorith,
  leading to recursion without control and severe out-of-memory conditions.

- Coordinate grid labels are no longer drawn or generated when clipped by the
  image bounding rectangle.

- Coordinate grid labels in right ascension are now generated as superscript
  time unit specifiers 'h', 'm' and 's' using the Unicode characters U+02B0,
  U+1D50 and U+02E2, respectively.

- The NamedStars catalog has now a maximum magnitude filter.

- PixInsight core version >= 1.8.8-9 required.
